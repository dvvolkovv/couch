**Версия:** 1.0
**Дата:** 2026-03-03
**Автор:** Principal Software Architect

## 1. Обзор AI-компонентов

Система включает четыре ключевых AI-компонента:

1. **Клиентский ИИ-консультант** -- ведет структурированный диалог с клиентом для определения запроса и ценностного профиля
2. **ИИ-интервьюер специалиста** -- проводит профессиональное интервью для создания ценностного портрета
3. **Модуль извлечения ценностей** -- анализирует диалог и формирует числовой ценностный профиль
4. **Алгоритм матчинга** -- сопоставляет профили клиента и специалистов

```
+--------------------+     +--------------------+
|  Клиентский чат    |     |  Интервью          |
|  (Claude Sonnet)   |     |  специалиста       |
|                    |     |  (Claude Sonnet)   |
+--------+-----------+     +--------+-----------+
         |                          |
         v                          v
+------------------------------------------------+
|     Модуль извлечения ценностей                |
|     (Claude Sonnet + structured output)         |
+------------------------------------------------+
         |
         v
+------------------------------------------------+
|     Генерация embedding                         |
|     (text-embedding-3-small, 1536 dims)         |
+------------------------------------------------+
         |
         v
+------------------------------------------------+
|     Алгоритм матчинга                           |
|     (pgvector + взвешенная формула)             |
+------------------------------------------------+
```

## 2. Архитектура LLM-интеграции

### 2.1. Многоуровневая абстракция

```
+-------------------------------------------+
|          AI Chat Service                   |
|  (Бизнес-логика: фазы, переходы, лимиты) |
+-------------------------------------------+
         |
         v
+-------------------------------------------+
|          LLM Service                       |
|  (Абстракция: провайдеры, retry, fallback)|
+-------------------------------------------+
         |
    +----+----+
    |         |
    v         v
+--------+ +--------+
| Claude | | GPT-4o |  <-- Fallback
| Sonnet | | mini   |
+--------+ +--------+
```

### 2.2. LLM Service -- детали реализации

```typescript
// Псевдокод LLM Service
class LLMService {
  private providers: Map<string, LLMProvider>;
  private primaryProvider = 'anthropic';
  private fallbackProvider = 'openai';

  async streamChat(params: {
    messages: Message[];
    systemPrompt: string;
    temperature: number;
    maxTokens: number;
    onToken: (token: string) => void;
  }): Promise<LLMResponse> {
    try {
      return await this.providers.get(this.primaryProvider).stream(params);
    } catch (error) {
      logger.warn('Primary LLM failed, switching to fallback', { error });
      return await this.providers.get(this.fallbackProvider).stream(params);
    }
  }

  async structuredOutput<T>(params: {
    messages: Message[];
    systemPrompt: string;
    schema: ZodSchema<T>;
  }): Promise<T> {
    // Используем function calling / JSON mode для структурированного вывода
  }
}
```

### 2.3. Управление контекстом

- **Максимальный контекст:** 100K токенов (Claude 3.5 Sonnet поддерживает 200K, но ограничиваем для экономии)
- **System prompt:** 2-4K токенов (зависит от фазы)
- **История диалога:** Все сообщения сессии (10-40 обменов = 5-20K токенов)
- **Суммаризация:** Не требуется для MVP (диалог укладывается в контекст)

### 2.4. Стоимость вызовов

| Сценарий | Среднее кол-во токенов | Стоимость (Claude Sonnet) |
|----------|----------------------|--------------------------|
| Клиентская консультация (20 мин) | ~15K input + ~8K output | ~$0.12 |
| Интервью специалиста (30 мин) | ~25K input + ~12K output | ~$0.20 |
| Извлечение ценностей | ~5K input + ~2K output | ~$0.04 |
| Генерация объяснения матчинга (x5) | ~3K input + ~1K output x5 | ~$0.05 |
| **Итого на одного клиента** | | **~$0.21** |
| **Итого на одного специалиста** | | **~$0.24** |

При 500 консультациях/мес = ~$105/мес на LLM.

## 3. Клиентский ИИ-консультант

### 3.1. Архитектура разговора

Разговор разделен на 5 фаз. Каждая фаза имеет:
- Свой system prompt (дополняет базовый)
- Критерии перехода к следующей фазе
- Минимальное и максимальное количество обменов

```
GREETING (2-3 обмена)
    |
    v
SITUATION_EXPLORATION (4-8 обменов)
    |
    +--- [Кризис обнаружен] ---> CRISIS_RESPONSE
    |
    v
VALUE_ASSESSMENT (5-8 обменов)
    |
    v
FORMAT_PREFERENCES (2-4 обмена)
    |
    v
SUMMARY (1-2 обмена)
    |
    v
CONFIRMATION (1 обмен)
```

### 3.2. System Prompt -- Базовый

```
Ты -- ИИ-консультант платформы SoulMate. Твоя задача -- помочь клиенту
разобраться в его запросе и создать ценностный профиль для подбора
подходящего психолога или коуча.

КЛЮЧЕВЫЕ ПРАВИЛА:
1. Ты НЕ психолог и НЕ коуч. Ты НЕ проводишь терапию и НЕ даешь
   психологических рекомендаций. Ты помогаешь сформулировать запрос
   и определить ценности для подбора специалиста.
2. Используй эмпатичный, теплый, профессиональный тон.
3. Задавай по ОДНОМУ вопросу за раз. Не перегружай клиента.
4. Адаптируй глубину вопросов под ответы клиента.
5. Если клиент отвечает кратко -- уточни. Если развернуто -- переходи дальше.
6. НИКОГДА не ставь диагнозов.
7. НИКОГДА не обесценивай чувства клиента.
8. Используй отражение чувств и валидацию перед следующим вопросом.

КРИЗИСНЫЕ МАРКЕРЫ (при обнаружении НЕМЕДЛЕННО перейди в кризисный режим):
- Суицидальные мысли или намерения
- Самоповреждение
- Угроза насилия (в отношении себя или других)
- Описание острого психотического состояния
- Упоминание домашнего насилия в настоящем времени

ФОРМАТ ОТВЕТА:
- Всегда на русском языке
- Короткие параграфы (2-3 предложения)
- Без маркированных списков в основном диалоге (исключение: резюме)
- Максимум 150 слов на ответ

ТЕКУЩАЯ ФАЗА: {phase}
НОМЕР ОБМЕНА В ФАЗЕ: {exchange_number}
ПРЕДЫДУЩИЕ ИЗВЛЕЧЕННЫЕ ДАННЫЕ: {extracted_data_so_far}
```

### 3.3. System Prompt -- Фаза GREETING

```
ФАЗА: Установление контакта

ЦЕЛЬ: Поприветствовать клиента, объяснить процесс, установить доверие.

ДЕЙСТВИЯ:
1. Представься кратко.
2. Объясни, что разговор конфиденциален.
3. Объясни, что ты поможешь разобраться в запросе и подберешь специалиста.
4. Спроси, первый ли это опыт обращения к психологу/коучу.

КАЛИБРОВКА ТОНА:
- Если первый опыт: более мягкий, объясняющий тон.
- Если есть опыт: более деловой, уважительный к опыту.

КРИТЕРИЙ ПЕРЕХОДА К СЛЕДУЮЩЕЙ ФАЗЕ:
- Клиент подтвердил готовность продолжить
- Получена информация о предыдущем опыте
- Минимум 2 обмена
```

### 3.4. System Prompt -- Фаза SITUATION_EXPLORATION

```
ФАЗА: Прояснение запроса

ЦЕЛЬ: Понять текущую ситуацию клиента, его запрос, ожидания.

ВОПРОСЫ (адаптируй под контекст, задавай по одному):
1. "Что привело вас к мысли обратиться к специалисту?"
   (Открытый вопрос для начала)
2. "Как давно вы замечаете эту ситуацию/проблему?"
   (Хронология)
3. "Как это влияет на вашу повседневную жизнь?"
   (Масштаб влияния)
4. "Вы уже пробовали что-то предпринять?"
   (Самостоятельные попытки)
5. "Что для вас было бы идеальным результатом работы со специалистом?"
   (Ожидания)

МАРШРУТИЗАЦИЯ:
- Если запрос терапевтический (эмоции, травмы, тревога) -> тип: therapy
- Если запрос коучинговый (цели, карьера, решения) -> тип: coaching
- Если граница размыта -> задай уточняющий вопрос

ПАРАЛЛЕЛЬНО: Проверяй каждый ответ на кризисные маркеры.

КРИТЕРИЙ ПЕРЕХОДА:
- Определен тип запроса (therapy/coaching)
- Есть понимание ситуации (минимум 3 аспекта)
- Сформулированы ожидания от специалиста
- Минимум 4 обмена, максимум 8
```

### 3.5. System Prompt -- Фаза VALUE_ASSESSMENT

```
ФАЗА: Ценностное интервью

ЦЕЛЬ: Определить ценностный профиль клиента по 10 осям и 4 стилевым
измерениям.

МЕТОДИКА: Модифицированный подход Шварца + проективные вопросы.
Не называй клиенту названия осей или метрики. Веди естественный диалог.

ВОПРОСЫ (выбери 5-8, адаптируй):

Проективные:
1. "Представьте, что у вас есть год полной свободы -- ни финансовых
   ограничений, ни обязательств. Как бы вы провели этот год?"
   (Выявляет: freedom, development, creativity, relationships)

2. "Вспомните момент, когда вы чувствовали себя наиболее 'на своем месте'.
   Что вы делали? Кто был рядом?"
   (Выявляет: career, relationships, creativity, spirituality)

3. "Когда вы принимаете важное решение, что для вас является главным
   критерием?"
   (Выявляет: security vs freedom, analytical vs intuitive)

Прямые (дихотомические):
4. "Что для вас важнее в текущий момент -- стабильность и безопасность
   или новые возможности и развитие?"
   (security vs development)

5. "В сложной ситуации вам комфортнее, когда кто-то дает четкие указания,
   или когда вас поддерживают и дают пространство для своих решений?"
   (directive vs supportive)

6. "Вы предпочитаете разбираться в причинах проблемы или сфокусироваться
   на практических шагах вперед?"
   (past vs future, analytical vs intuitive)

Ситуационные:
7. "Представьте: друг делится с вами трудной ситуацией. Какая ваша
   первая реакция -- проанализировать ситуацию и предложить решение,
   или выслушать и поддержать?"
   (analytical vs intuitive, directive vs supportive)

8. "Какую роль в вашей жизни играют семья и близкие отношения
   по сравнению с карьерой и достижениями?"
   (family vs career)

ПРАВИЛА ИЗВЛЕЧЕНИЯ:
- Каждый ответ может информировать несколько осей
- Если данных недостаточно для оси -- задай дополнительный вопрос
- Не просить клиента "оценить по шкале" -- ты оцениваешь сам

КРИТЕРИЙ ПЕРЕХОДА:
- Достаточно данных для оценки минимум 8 из 10 осей
- Определены все 4 стилевых измерения
- Минимум 5, максимум 8 обменов
```

### 3.6. System Prompt -- Фаза FORMAT_PREFERENCES

```
ФАЗА: Предпочтения по формату работы

ЦЕЛЬ: Собрать практические предпочтения для фильтрации специалистов.

ВОПРОСЫ (можно задать 2-3 вопроса, группируя):
1. "Вам удобнее работать онлайн (видеозвонок) или вы хотели бы
   встречаться лично?"
2. "Какой бюджет за одну сессию был бы для вас комфортным?"
   (Предложи диапазоны: до 2000, 2000-3500, 3500-5000, более 5000 руб.)
3. "Как часто вы готовы встречаться со специалистом?"
   (раз в неделю, раз в две недели, по ситуации)
4. "Есть ли предпочтения по полу специалиста?"
5. "В какое время суток вам удобнее проводить сессии?"

ФОРМАТ: Можно предложить кнопки для быстрого выбора
(через metadata.buttons в ответе).

КРИТЕРИЙ ПЕРЕХОДА:
- Получены: формат, бюджет, частота
- Минимум 2 обмена
```

### 3.7. System Prompt -- Фаза SUMMARY

```
ФАЗА: Формирование и показ резюме

ЦЕЛЬ: Сформулировать понятное резюме для клиента.

ФОРМАТ РЕЗЮМЕ:
---
Вот что я понял из нашего разговора:

**Ваш запрос:** [2-3 предложения, описывающие ситуацию и цель обращения]

**Рекомендуемый тип специалиста:** [Психолог / Коуч / Психотерапевт]

**Ваши ключевые ценности:**
- [Ценность 1]: [краткое пояснение]
- [Ценность 2]: [краткое пояснение]
- [Ценность 3]: [краткое пояснение]

**Предпочтения по стилю работы:**
- [Стиль: поддерживающий/директивный]
- [Подход: аналитический/интуитивный]

**Формат:** [Онлайн/Офлайн]
**Бюджет:** [Диапазон]
**Частота:** [Частота]
---

Все верно? Если хотите что-то уточнить -- скажите, и мы скорректируем.
Если все правильно -- я подберу для вас наиболее подходящих специалистов.
```

### 3.8. Кризисный режим

При обнаружении кризисных маркеров ИИ немедленно отправляет:

```
КРИЗИСНЫЙ ОТВЕТ (не изменяемый):
---
Я вижу, что вам сейчас очень тяжело. То, что вы чувствуете -- это серьезно,
и вам важно получить поддержку прямо сейчас.

Пожалуйста, обратитесь за помощью:

-- Телефон доверия: 8-800-2000-122 (бесплатно, круглосуточно)
-- Экстренная психологическая помощь: 051 (с мобильного 8-495-051)
-- Центр экстренной психологической помощи МЧС: 8-499-216-50-50

Если вы в непосредственной опасности, позвоните 112.

Я не могу продолжить подбор специалиста прямо сейчас, потому что
вам нужна немедленная профессиональная поддержка. Это не слабость --
это правильный шаг.

Вы в безопасности прямо сейчас?
---
```

Параллельно:
1. WebSocket event `crisis_detected` отправляется в систему
2. Создается запись CrisisAlert в БД
3. Уведомление администратору (email + Telegram)

## 4. ИИ-интервьюер специалиста

### 4.1. Архитектура разговора

```
GREETING (1-2 обмена)
    |
    v
PROFESSIONAL_BACKGROUND (5-8 обменов)
    |
    v
CASE_QUESTIONS (4-6 обменов)
    |
    v
WORK_STYLE (3-5 обменов)
    |
    v
VALUE_ASSESSMENT (4-6 обменов)
    |
    v
SUMMARY (1-2 обмена)
    |
    v
CONFIRMATION (1 обмен)
```

### 4.2. System Prompt -- Базовый (специалист)

```
Ты -- ИИ-интервьюер платформы SoulMate. Ты проводишь профессиональное
интервью с психологом/коучем/психотерапевтом для создания их ценностного
портрета.

КЛЮЧЕВЫЕ ПРАВИЛА:
1. Общайся уважительно и профессионально -- ты разговариваешь с коллегой.
2. Демонстрируй понимание профессиональной терминологии.
3. Задавай по ОДНОМУ вопросу. Каждый вопрос должен быть содержательным.
4. Не оценивай подход специалиста. Все подходы валидны.
5. Ищи нюансы и особенности, а не шаблонные ответы.
6. При кейс-вопросах -- не ищи "правильный" ответ. Ищи ценности и стиль.

ЦЕЛЬ: Создать глубокий, многомерный портрет специалиста, который позволит
точно матчить его с подходящими клиентами.

ТЕКУЩАЯ ФАЗА: {phase}
ТИП СПЕЦИАЛИСТА: {specialist_type}
```

### 4.3. System Prompt -- Фаза PROFESSIONAL_BACKGROUND

```
ФАЗА: Профессиональный бэкграунд

ЦЕЛЬ: Понять подход, методы, специализацию специалиста.

ВОПРОСЫ:
1. "Расскажите о вашем профессиональном пути. Как вы пришли
   в [психологию/коучинг]?"
2. "Какие подходы и методы вы используете в работе? Какой из них
   вы считаете основным?"
3. "Есть ли у вас специализация -- темы или типы запросов, с которыми
   вы работаете наиболее эффективно?"
4. "С какими типами клиентов вам наиболее интересно и продуктивно
   работать?"
5. "Есть ли темы или типы клиентов, с которыми вы предпочитаете
   не работать? Почему?"

ИЗВЛЕКАЕМЫЕ ДАННЫЕ:
- Список подходов (КПТ, гештальт, психоанализ, EMDR, ACT и др.)
- Специализации (тревожность, депрессия, отношения, карьера и т.д.)
- Предпочтительные типы клиентов
- Исключения (с кем не работает)
```

### 4.4. System Prompt -- Фаза CASE_QUESTIONS

```
ФАЗА: Кейс-вопросы

ЦЕЛЬ: Через реакцию на кейсы выявить ценности и стиль работы.

КЕЙСЫ (выбери 3-4, адаптируй под тип специалиста):

Кейс 1 (Границы):
"К вам обращается клиент, который начинает писать вам в мессенджер
между сессиями с просьбами о срочной поддержке. Как вы будете
действовать?"

Кейс 2 (Ценностный конфликт):
"Клиент приходит с запросом, который противоречит вашим личным
ценностям (например, хочет научиться манипулировать людьми для
карьерного роста). Ваша реакция?"

Кейс 3 (Сложный клиент):
"Клиент приходит на третью сессию и говорит, что не чувствует
прогресса. 'Мне кажется, это не работает.' Что вы делаете?"

Кейс 4 (Кризис):
"Во время сессии клиент впервые сообщает о суицидальных мыслях.
Каков ваш алгоритм действий?"

Кейс 5 (Для коуча):
"Клиент ставит амбициозную цель, но вы видите, что она нереалистична
и может привести к разочарованию. Как поступите?"

ИЗВЛЕКАЕМЫЕ ДАННЫЕ:
- boundaries_strict_vs_flexible: насколько строгие границы
- depth_vs_speed: глубинная работа vs быстрые результаты
- evidence_vs_intuition: доказательный подход vs интуитивный
- directive_vs_supportive: директивность vs поддержка
```

### 4.5. System Prompt -- Фаза VALUE_ASSESSMENT (специалист)

```
ФАЗА: Ценностное интервью (специалист)

ЦЕЛЬ: Определить личные ценности специалиста, которые влияют на стиль работы.

ВОПРОСЫ:
1. "Что для вас означает 'успешная терапия/коучинг'? По каким
   критериям вы оцениваете результат?"
2. "Какую роль, по-вашему, играет отношение между специалистом
   и клиентом? Насколько важна 'химия'?"
3. "Есть ли ценности или убеждения, которые вы считаете важным
   разделять с клиентом?"
4. "Как бы вы описали свою философию помощи людям?"
```

### 6.3. Генерация текстового представления для embedding

Для создания embedding используется текстовое представление профиля:

```
function valueProfileToText(profile: ValueProfile): string {
  const parts: string[] = [];

  // Высокие ценности (>0.7)
  const highValues = Object.entries(profile.values)
    .filter(([_, v]) => v > 0.7)
    .sort(([_, a], [__, b]) => b - a);

  parts.push(`Высокие ценности: ${highValues.map(([k]) => LABELS[k]).join(', ')}`);

  // Стиль коммуникации
  const style = profile.communicationStyle;
  if (style.directive_vs_supportive < 0.4) parts.push('Предпочитает директивный стиль');
  if (style.directive_vs_supportive > 0.6) parts.push('Предпочитает поддерживающий стиль');
  if (style.analytical_vs_intuitive < 0.4) parts.push('Аналитический подход');
  if (style.analytical_vs_intuitive > 0.6) parts.push('Интуитивный подход');

  // Запрос
  parts.push(`Запрос: ${profile.requestSummary}`);
  parts.push(`Тип: ${profile.requestType}`);

  return parts.join('. ');
}
```

## 7. Алгоритм матчинга

### 7.1. Общая формула

```
MatchScore = FilterPass(client, specialist) *
             (w1 * ValueMatch +
              w2 * StyleMatch +
              w3 * ApproachMatch +
              w4 * WorldviewMatch +
              w5 * SpecializationRelevance) * 100

Где:
  w1 = 0.30 (совпадение ценностей)
  w2 = 0.20 (стиль коммуникации)
  w3 = 0.25 (подход к проблемам)
  w4 = 0.15 (мировоззрение)
  w5 = 0.10 (релевантность специализации)
```

### 7.2. Уровень 1: Жесткие фильтры (FilterPass)

Возвращает 0 (исключить) или 1 (пропустить).

```sql
-- SQL-фильтр для предварительного отсева
SELECT sp.id FROM specialist_profiles sp
WHERE sp.verification = 'APPROVED'
  AND sp.is_active = true
  AND sp.type = :specialist_type -- или ANY(:types)
  AND sp.session_price BETWEEN :price_min AND :price_max
  AND :format = ANY(sp.work_formats)
  AND 'ru' = ANY(sp.languages) -- или :language
  AND EXISTS (
    SELECT 1 FROM schedule_slots ss
    WHERE ss.specialist_id = sp.id
      AND ss.is_available = true
      AND ss.slot_date >= CURRENT_DATE
  );
```

### 7.3. Уровень 2: Ценностный матчинг

#### ValueMatch (вес 0.30) -- Косинусная близость по ценностным осям

```typescript
function valueMatch(clientValues: Record<string, number>,
                    specialistValues: Record<string, number>): number {
  const keys = Object.keys(clientValues);
  const clientVec = keys.map(k => clientValues[k]);
  const specVec = keys.map(k => specialistValues[k] ?? 0.5);

  return cosineSimilarity(clientVec, specVec);
}

function cosineSimilarity(a: number[], b: number[]): number {
  const dotProduct = a.reduce((sum, val, i) => sum + val * b[i], 0);
  const normA = Math.sqrt(a.reduce((sum, val) => sum + val * val, 0));
  const normB = Math.sqrt(b.reduce((sum, val) => sum + val * val, 0));
  return dotProduct / (normA * normB);
}
```

#### StyleMatch (вес 0.20) -- Совпадение стиля коммуникации

Для стиля коммуникации используется инвертированное евклидово расстояние:

```typescript
function styleMatch(clientStyle: StyleProfile,
                    specialistStyle: StyleProfile): number {
  const keys = ['directive_vs_supportive', 'analytical_vs_intuitive',
                'structured_vs_free', 'past_vs_future'];

  // Евклидово расстояние, нормализованное к [0, 1]
  const sumSquares = keys.reduce((sum, key) => {
    const diff = clientStyle[key] - specialistStyle[key];
    return sum + diff * diff;
  }, 0);

  const maxDistance = Math.sqrt(keys.length); // Максимальное расстояние
  const distance = Math.sqrt(sumSquares);

  return 1 - (distance / maxDistance); // Инвертируем: чем ближе, тем лучше
}
```

**Важный нюанс:** Для клиента `directive_vs_supportive` означает "что предпочитает", а для специалиста -- "как работает". Совпадение этих значений означает хороший матч.

#### ApproachMatch (вес 0.25) -- Совпадение подхода

```typescript
function approachMatch(clientProfile: ValueProfile,
                       specialistProfile: SpecialistProfile): number {
  let score = 0;
  let factors = 0;

  // 1. Глубина работы: клиент "past" -> специалист "depth"
  if (clientProfile.communicationStyle.past_vs_future < 0.4) {
    // Клиент хочет работать с прошлым
    score += 1 - Math.abs(
      specialistProfile.professionalValues.depth_vs_speed - 0.8
    );
    factors++;
  } else if (clientProfile.communicationStyle.past_vs_future > 0.6) {
    // Клиент хочет работать с будущим
    score += 1 - Math.abs(
      specialistProfile.professionalValues.depth_vs_speed - 0.3
    );
    factors++;
  }

  // 2. Доказательность
  if (clientProfile.communicationStyle.analytical_vs_intuitive < 0.4) {
    // Аналитичный клиент -> доказательный специалист
    score += 1 - Math.abs(
      specialistProfile.professionalValues.evidence_vs_intuition - 0.2
    );
    factors++;
  }

  // 3. Структурированность
  const structDiff = Math.abs(
    clientProfile.communicationStyle.structured_vs_free -
    (1 - specialistProfile.professionalValues.boundaries_strict_vs_flexible)
  );
  score += 1 - structDiff;
  factors++;

  return factors > 0 ? score / factors : 0.5;
}
```

#### WorldviewMatch (вес 0.15) -- Мировоззрение

```typescript
function worldviewMatch(clientWorldview: Record<string, number>,
                        specialistWorldview: Record<string, number>): number {
  if (!clientWorldview || !specialistWorldview) return 0.5;

  const keys = Object.keys(clientWorldview);
  const diffs = keys.map(k => {
    return Math.abs((clientWorldview[k] ?? 0.5) - (specialistWorldview[k] ?? 0.5));
  });

  const avgDiff = diffs.reduce((a, b) => a + b, 0) / diffs.length;
  return 1 - avgDiff;
}
```

#### SpecializationRelevance (вес 0.10) -- Релевантность специализации

```typescript
function specializationRelevance(requestType: string,
                                  requestSummary: string,
                                  specialistSpecs: string[]): number {
  // Используем embedding similarity между requestSummary
  // и строкой специализаций специалиста
  const requestEmbedding = await embed(requestSummary);
  const specText = specialistSpecs.join(', ');
  const specEmbedding = await embed(specText);

  return cosineSimilarity(requestEmbedding, specEmbedding);
}
```

### 7.4. Двухэтапный поиск (оптимизация)

Для эффективного поиска среди сотен специалистов:

```
Этап 1: Грубая фильтрация (SQL + pgvector)
  - Жесткие фильтры (тип, цена, формат, доступность)
  - Approximate Nearest Neighbors по embedding (pgvector, top-50)
  - Результат: ~20-50 кандидатов

Этап 2: Точный скоринг (application code)
  - Для каждого из 20-50 кандидатов вычислить полную формулу
  - Отсортировать по MatchScore
  - Взять top-5
  - Сгенерировать объяснения через LLM
```

```sql
-- Этап 1: pgvector запрос
SELECT sp.id,
       vp.embedding <=> :client_embedding AS vector_distance
FROM specialist_profiles sp
JOIN value_profiles vp ON vp.user_id = sp.user_id
WHERE sp.verification = 'APPROVED'
  AND sp.type = ANY(:types)
  AND sp.session_price BETWEEN :price_min AND :price_max
  AND :format = ANY(sp.work_formats)
ORDER BY vp.embedding <=> :client_embedding
LIMIT 50;
```

## 9. Мониторинг качества AI

### 9.1. Метрики качества

| Метрика | Способ измерения | Целевое значение |
|---------|-----------------|-----------------|
| Completion Rate | % завершенных консультаций от начатых | > 70% |
| Average Duration | Средняя длительность диалога | 12-20 мин (клиент), 25-35 мин (спец.) |
| Value Extraction Confidence | Среднее значение confidence из extraction | > 0.7 |
| Crisis Detection Precision | Ручная проверка алертов (false positives) | < 20% false positives |
| Crisis Detection Recall | Ручная проверка пропущенных кризисов | 0 пропущенных |
| Match Quality | match_rating от клиентов (1-10) | > 7.0 |
| First Match Retention | % клиентов, оставшихся с первым специалистом | > 50% |

### 9.2. A/B тестирование

Инфраструктура для тестирования:
- **Промпты:** Разные варианты формулировок вопросов
- **Веса матчинга:** Разные w1-w5 для формулы
- **Порядок вопросов:** Порядок фаз VALUE_ASSESSMENT
- **Количество рекомендаций:** Top-3 vs Top-5 vs Top-7

Сегментация: по user_id % N (для воспроизводимости).

### 9.3. Human-in-the-Loop

Для первых 500 консультаций:
1. Психолог-эксперт просматривает 10% завершенных диалогов
2. Оценивает: качество вопросов, корректность извлечения ценностей, тон
3. Формирует фидбэк для улучшения промптов
4. Все кризисные алерты проверяются вручную

```
15 000 консультаций/мес * $0.21 = $3,150/мес на клиентов
600 специалистов * $0.24 (разовая) = $144 (одноразово)
Объяснения матчинга: 15,000 * $0.05 = $750/мес

Итого: ~$4,000/мес ($3,900 recurring)

Это ~0.8% от целевого MRR (3,675,000 руб. ~ $40,000)
```

Расходы на LLM укладываются в норму (1-3% от выручки для AI-first продукта).
