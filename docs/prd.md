## 1. Резюме проекта (Executive Summary)

SoulMate -- маркетплейс-платформа для подбора психологов и коучей с уникальным ценностно-ориентированным подходом к матчингу. В отличие от существующих решений (Ясно, Alter, Zigmund.online), которые подбирают специалистов по формальным критериям (специализация, цена, время), SoulMate использует ИИ-агентов для глубинного определения ценностей, мировоззрения и подходов как клиентов, так и специалистов, обеспечивая качественно иной уровень совпадения.

**Ключевая гипотеза:** Совпадение ценностей и мировоззрения между клиентом и специалистом является критическим фактором терапевтического альянса. Платформа, которая обеспечивает этот уровень матчинга, даст значимо лучшие результаты терапии и, как следствие, более высокую retention rate.

**Целевой рынок:** Российский рынок психологических услуг -- 49.7 млрд руб. (2024), с ростом 25-28% г/г. Онлайн-сегмент растет опережающими темпами (мобильный трафик +116% в 2024).

## 2. Проблема (Problem Statement)

### Проблемы клиентов

1. **Непонимание собственного запроса.** Большинство людей, впервые обращающихся к психологу, не могут четко сформулировать свой запрос. Существующие платформы предлагают анкету из 5-10 вопросов, что недостаточно.

2. **Формальный подбор.** Текущие сервисы подбирают по специализации, цене и времени. Это как подбирать друга по росту и весу. Ценностное совпадение -- ключевой предиктор успешной терапии -- игнорируется.

3. **Высокий процент смены специалиста.** По данным отрасли, 30-40% клиентов меняют первого подобранного специалиста. Каждая неудачная попытка -- это потерянные деньги, время и мотивация продолжать терапию.

4. **Барьер первого шага.** Стигма, страх, непонимание процесса -- все это мешает людям обратиться за помощью. Разговор с ИИ снижает этот барьер.

### Проблемы специалистов

1. **Нецелевые клиенты.** Специалисты тратят первые 1-3 сессии на прояснение запроса, который мог бы быть проработан до начала терапии.

2. **Ценностные конфликты.** Несовпадение мировоззрения приводит к разрыву терапевтического альянса и оттоку клиентов.

3. **Конкуренция по цене.** На существующих площадках специалисты конкурируют в основном по цене, а не по качеству и уникальности подхода.

## 3. Целевая аудитория

### Клиенты (B2C)

- **Возраст:** 25-45 лет
- **География:** Россия (Москва, Санкт-Петербург, города-миллионники), русскоязычная диаспора
- **Доход:** Средний и выше среднего (готовность платить 2000-6000 руб./сессию)
- **Сегменты:**
  - Первично обращающиеся (никогда не были у психолога) -- 40%
  - Неудовлетворенные текущим специалистом -- 30%
  - Ищущие коуча для профессионального развития -- 20%
  - Возвращающиеся после перерыва -- 10%

### Специалисты (B2B)

- **Психологи** с высшим образованием и практикой от 2 лет
- **Коучи** с сертификацией (ICF, EMCC или аналоги)
- **Психотерапевты** с медицинским образованием
- **Сегменты по мотивации:**
  - Начинающие, ищущие первых клиентов -- 25%
  - Опытные, расширяющие частную практику -- 40%
  - Эксперты, ищущие целевых "своих" клиентов -- 35%

### 4.4. Ценностный матчинг (Value-Based Matching)

**Описание:** Алгоритм подбора специалиста на основе многомерного совпадения ценностей, запроса и предпочтений.

**Концепция алгоритма:**

Матчинг строится на трех уровнях:

**Уровень 1 -- Жесткие фильтры (обязательное совпадение):**
- Тип специалиста (психолог/коуч/психотерапевт)
- Формат работы (онлайн/офлайн/гибрид)
- Ценовой диапазон
- Доступность в расписании
- Язык общения

**Уровень 2 -- Ценностный матчинг (взвешенная оценка):**
- Совпадение по осям ценностей (семья, карьера, свобода, безопасность, развитие, духовность и др.) -- вес 30%
- Совпадение стиля коммуникации (директивный/поддерживающий) -- вес 20%
- Совпадение по подходу к решению проблем (аналитический/интуитивный, краткосрочный/глубинный) -- вес 25%
- Совпадение мировоззренческих установок -- вес 15%
- Релевантность специализации запросу клиента -- вес 10%

**Уровень 3 -- Поведенческая оптимизация:**
- Учет исторических данных об успешных матчах (retention, отзывы)
- A/B тестирование весов факторов
- Collaborative filtering (клиенты с похожим профилем успешно работали с данным специалистом)

**Требования:**

| ID | Требование | Приоритет |
|----|-----------|-----------|
| MTH-01 | Алгоритм выдает Top-5 рекомендованных специалистов с процентом совпадения | Must |
| MTH-02 | Для каждого рекомендованного специалиста показывается объяснение: "почему подходит" | Must |
| MTH-03 | Клиент может указать, что рекомендация не подошла, с указанием причины | Must |
| MTH-04 | Алгоритм обучается на данных обратной связи | Should |
| MTH-05 | Время генерации рекомендаций -- не более 5 секунд | Must |
| MTH-06 | При недостатке данных для матчинга (мало специалистов) -- показывается предупреждение | Must |
| MTH-07 | Визуализация совпадения: радарная диаграмма или аналог | Should |

## 5. Пользовательские потоки (User Flows)

### 5.1. Поток клиента

```
1. Посещение лендинга/главной страницы
2. Регистрация (email / телефон / Google / VK ID)
3. Запуск ИИ-консультации
   3.1. ИИ приветствует и объясняет процесс
   3.2. Серия вопросов о ситуации и запросе (5-7 мин)
   3.3. Серия вопросов для определения ценностного профиля (5-10 мин)
   3.4. Определение предпочтений по формату работы (2-3 мин)
   3.5. Формирование и показ резюме клиенту
   3.6. Клиент подтверждает или корректирует
4. Получение рекомендаций (Top-5 специалистов)
5. Просмотр профилей рекомендованных специалистов
6. Выбор специалиста
7. Бронирование первой сессии
8. Оплата
9. Проведение сессии
10. Оставление отзыва
11. Повторное бронирование или запрос нового подбора
```

### 5.2. Поток специалиста

```
1. Заявка на регистрацию (базовые данные, загрузка документов)
2. ИИ-интервью
   2.1. Вводная часть: объяснение процесса и целей
   2.2. Профессиональный блок: образование, подходы, методы
   2.3. Ценностный блок: кейсы, проективные вопросы, ситуации
   2.4. Стиль работы: предпочтения по формату, типы клиентов
   2.5. Формирование ценностного портрета
   2.6. Специалист просматривает и подтверждает портрет
3. Верификация (модерация документов)
4. Настройка профиля (фото, видео-визитка, расписание, цены)
5. Активация профиля
6. Получение запросов от клиентов
7. Проведение сессий
8. Получение оплаты
```

### 5.3. Детальный поток ИИ-консультации клиента

**Фаза 1: Установление контакта (2-3 минуты)**
- Приветствие, объяснение конфиденциальности
- Уточнение, первый ли это опыт обращения к специалисту
- Калибровка тона (более мягкий для новичков, более деловой для опытных)

**Фаза 2: Прояснение запроса (5-7 минут)**
- Открытый вопрос: "Что привело вас сюда?"
- Уточняющие вопросы по ситуации
- Определение типа запроса (терапевтический, коучинговый, кризисный)
- Проверка на кризисные маркеры (суицидальные мысли, насилие, острые состояния)
- При обнаружении кризиса -- немедленное перенаправление на горячую линию

**Фаза 3: Ценностное интервью (5-10 минут)**
- Методика: модифицированный опросник Шварца + проективные вопросы
- Вопросы типа: "Что для вас означает успешная жизнь?", "Опишите ситуацию, когда вы чувствовали себя наиболее подлинным"
- Определение по 8-12 осям ценностей с нормализацией
- Выявление предпочтений: жесткая обратная связь vs мягкая поддержка, структура vs свобода, фокус на прошлом vs будущем

**Фаза 4: Предпочтения по формату (2-3 минуты)**
- Онлайн/офлайн/гибрид
- Частота сессий
- Ценовой диапазон
- Пол специалиста (если есть предпочтение)
- Время суток

**Фаза 5: Резюме и подтверждение (2-3 минуты)**
- ИИ формулирует краткое резюме запроса
- Показывает ценностный профиль в визуальной форме
- Клиент подтверждает или вносит корректировки
- Переход к рекомендациям

## 6. Модель монетизации

### Анализ вариантов

| Модель | Плюсы | Минусы | Оценка |
|--------|-------|--------|--------|
| **Комиссия с каждой сессии (15-25%)** | Выручка масштабируется с GMV; низкий барьер входа для специалистов | Мотивирует уводить клиентов с платформы; высокая ставка отпугивает топ-специалистов | 6/10 |
| **Подписка для специалистов (фикс/мес)** | Предсказуемая выручка; специалисты не мотивированы уводить клиентов | Высокий барьер входа; не масштабируется с ростом GMV | 5/10 |
| **Гибрид: подписка + сниженная комиссия** | Базовый доход + масштабируемость; гибкость | Сложность объяснения модели | 7/10 |
| **Freemium для клиентов + комиссия** | Низкий барьер для клиентов; конверсия в premium | Нужно определить ценность premium; может не конвертироваться | 7/10 |
| **Комиссия + Premium-подписка для клиентов** | Два потока дохода; ценность для обеих сторон | Сложность; риск отторжения клиентами | 8/10 |

### Рекомендуемая модель: Гибридная (Commission + Freemium)

**Для специалистов:**
- **Базовый профиль (бесплатно):** Прохождение ИИ-интервью, создание профиля, до 5 клиентов через платформу в месяц, комиссия 20% с каждой сессии
- **Профессионал (2 990 руб./мес):** Неограниченное количество клиентов, комиссия снижена до 12%, приоритет в выдаче, расширенная аналитика
- **Эксперт (5 990 руб./мес):** Все из "Профессионал" + комиссия 8%, бейдж "Топ-специалист", участие в продвижении, персональный менеджер

**Для клиентов:**
- **Бесплатно:** ИИ-консультация, просмотр каталога, 1 рекомендация Top-5
- **Premium (499 руб./мес):** Неограниченные ИИ-консультации, расширенные рекомендации, приоритетное бронирование, трекер прогресса
- **Первая ИИ-консультация всегда бесплатна** -- ключевое для привлечения

**Прогноз unit economics (на 12 месяц):**
- Средняя стоимость сессии: 3 500 руб.
- Средняя комиссия: 15% = 525 руб./сессия
- Целевое количество сессий/мес (12 мес): 5 000
- Доход от комиссий: 2 625 000 руб./мес
- Доход от подписок специалистов (200 платных): 800 000 руб./мес
- Доход от подписок клиентов (500 платных): 250 000 руб./мес
- **Целевой MRR (12 мес): 3 675 000 руб./мес**

## 8. MVP vs. Будущие фазы

### MVP (Phase 1) -- 3-4 месяца разработки

**В скоуп входит:**
- Регистрация и авторизация (email + телефон)
- ИИ-консультация клиента (текстовый чат, базовая версия)
- ИИ-интервью специалиста (текстовый чат, базовая версия)
- Каталог специалистов с ценностным профилем
- Ценностный матчинг (базовый алгоритм, уровни 1 и 2)
- Бронирование через календарь
- Интеграция с внешними видео-сервисами (Zoom/Google Meet)
- Оплата через платежный агрегатор (ЮKassa)
- Базовая админ-панель
- Веб-приложение (responsive)

**Не входит в MVP:**
- Мобильное приложение
- Встроенный видеочат
- Голосовой ввод в ИИ-консультации
- Поведенческая оптимизация матчинга (уровень 3)
- Подарочные сертификаты и рассрочка
- B2B (корпоративные клиенты)
- Международная версия

### Phase 2 -- 3-4 месяца после запуска

- Мобильные приложения (iOS, Android)
- Встроенный видеочат
- Расширенная аналитика для специалистов
- Трекер прогресса для клиентов
- Collaborative filtering в матчинге
- Внутренний мессенджер
- Реферальная программа
- SEO-контент и блог

### Phase 3 -- 6+ месяцев после запуска

- B2B-продукт (корпоративная психологическая поддержка)
- Международная локализация (английский, казахский, узбекский)
- Голосовой ИИ-агент
- Групповые сессии
- Курсы и вебинары от специалистов (marketplace контента)
- Интеграция с медицинскими системами

## 9. Риски и митигации

### Высокие риски

| Риск | Вероятность | Влияние | Митигация |
|------|------------|---------|-----------|
| **Недостаток специалистов на старте** (cold start problem) | Высокая | Критическое | Ручной рекрутинг 50-100 специалистов до запуска; бесплатный период для первых 100 специалистов; партнерства с профессиональными ассоциациями |
| **Низкое качество ИИ-консультации** -- клиенты не доверяют ИИ или ИИ дает неадекватные рекомендации | Средняя | Критическое | Итеративная доработка промптов с привлечением практикующих психологов; A/B тестирование; человеческая модерация первых 500 консультаций; явный disclaimer что ИИ не заменяет специалиста |
| **Регуляторные ограничения** -- ФЗ о персональных данных, медицинское лицензирование | Средняя | Высокое | Юридический аудит до запуска; хранение данных на территории РФ; четкое позиционирование как информационный сервис, а не медицинский |
| **Уход клиентов "мимо платформы"** после первого матча | Высокая | Высокое | Ценность платформы за пределами матчинга (трекер, аналитика, удобство оплаты); умеренная комиссия; пакеты сессий |

### Средние риски

| Риск | Вероятность | Влияние | Митигация |
|------|------------|---------|-----------|
| **Конкуренты скопируют ИИ-подход** | Средняя | Среднее | Скорость вывода на рынок; накопление данных для матчинга (moat); глубина ИИ-интервью как конкурентное преимущество |
| **Высокий CAC** из-за конкуренции в маркетинге | Высокая | Среднее | Контент-маркетинг; SEO; реферальная программа; партнерства |
| **Этические вопросы** -- ИИ "ставит диагнозы" или дает вредные советы | Средняя | Высокое | ИИ не ставит диагнозов; четкие guardrails; обязательная передача кризисных кейсов людям; этический комитет |

### Допущения (Assumptions)

1. Ценностное совпадение действительно повышает качество терапевтического альянса (нужна валидация)
2. Клиенты готовы проводить 15-20 минут в диалоге с ИИ перед подбором
3. Специалисты готовы проходить 30-40 минутное ИИ-интервью
4. Российский рынок онлайн-терапии продолжит расти темпами 20%+ в год
5. Текущие LLM-модели достаточно качественны для проведения ценностного интервью

### Зависимости (Dependencies)

1. LLM API (OpenAI / Anthropic / YandexGPT) -- доступность и стоимость
2. Платежный агрегатор (ЮKassa / CloudPayments)
3. Видеосервис (Zoom API / собственное решение)
4. Хостинг в РФ (Yandex Cloud / Selectel) для соответствия ФЗ-152
5. Привлечение экспертов-психологов для валидации ИИ-интервью

## 10. Технические ограничения и соответствие

### Соответствие законодательству

- **ФЗ-152 "О персональных данных":** Все данные хранятся на территории РФ. Согласие на обработку ПД при регистрации.
- **ФЗ-149 "Об информации":** Платформа позиционируется как информационный сервис, а не медицинское учреждение.
- **Медицинское лицензирование:** Не требуется, так как платформа не оказывает медицинские услуги, а обеспечивает связь клиента и независимого специалиста.
- **ФЗ-54 "О ККТ":** Интеграция с онлайн-кассой для фискализации.

### Безопасность данных

- Шифрование данных at rest и in transit (AES-256, TLS 1.3)
- Данные ИИ-консультаций -- особая категория, хранение с повышенной защитой
- Право на удаление данных (GDPR-подобное для РФ)
- Двухфакторная аутентификация для специалистов
- Логирование доступа к чувствительным данным

## Приложение: Альтернативные названия

1. **SoulMate** -- основной вариант. Передает идею "родственной души" в контексте подбора. Риск: может ассоциироваться с дейтингом.
2. **ValuMatch** -- прямое указание на ценностный матчинг. Менее эмоциональное.
3. **InSync** -- "в синхронности". Подходит для международного рынка.
4. **TerraVita** -- "земля жизни". Абстрактное, но запоминающееся.
5. **MyMind** -- простое, понятное. Может быть занято.
